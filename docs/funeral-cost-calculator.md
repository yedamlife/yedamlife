# 장례비용 알아보기 — 기획서

## 1. 개요

사용자가 장례형태, 지역, 장례식장을 선택하면 **실제 데이터 기반의 예상 장례비용**을 산출하고, 예담라이프 상품과 비교하여 절약 효과를 체감할 수 있는 인터랙티브 시뮬레이터.

### 목표

- 장례비용에 대한 막연한 불안감 해소
- 실제 장례식장 가격 데이터 기반 신뢰도 확보
- 자연스러운 예담 상품 추천 연결

### 진입점

- 후불제 상조 히어로 섹션 "장례비용 알아보기" 버튼
- **풀스크린 모달**로 전개 (배경 딤 처리, 스크롤 잠금)

## 3. 각 Step 상세

### Step 1. 장례형태 선택

가장 먼저 장례 형태를 결정하여 이후 플로우 분기를 확정한다.

| 항목 | 설명 |
|------|------|
| UI | 2개 카드 택 1 (라디오 스타일) |
| 옵션 | **3일장** / **무빈소** |
| 동작 | 선택 즉시 Step 2 진입 |

```
┌──────────────────────┐  ┌──────────────────────┐
│        3일장           │  │       무빈소           │
│                       │  │                      │
│   빈소를 마련하여       │  │   빈소 없이            │
│   3일간 조문 진행       │  │   당일 장례 진행        │
│                       │  │                      │
│   · 일반적인 장례       │  │   · 소규모/간소 장례    │
│   · 조문객 50명 이상    │  │   · 가족장/직계 중심    │
└──────────────────────┘  └──────────────────────┘
```

**선택에 따른 이후 플로우:**
- **3일장** → Step 2 → Step 3 → Step 4 (빈소 규모 + 조문객 입력) → Step 5
- **무빈소** → Step 2 → Step 3 → Step 5 (Step 4 스킵, 시설 사용료 제외)

### Step 2. 지역 선택

| 항목 | 설명 |
|------|------|
| UI | 시도 Select → 시군구 Select (연동) |
| 데이터 | `funeral_halls` 테이블의 `sido_cd` / `gungu_cd` 활용 |
| 동작 | 시군구 선택 완료 시 자동으로 Step 3 진입 |

**시도 목록 추출 로직:**
```sql
SELECT DISTINCT sido_cd,
  SPLIT_PART(org_name, ' ', 1) AS sido_name
FROM funeral_halls
ORDER BY sido_name;
```

**시군구 목록:**
```sql
SELECT DISTINCT gungu_cd, org_name
FROM funeral_halls
WHERE sido_cd = :selected_sido
ORDER BY org_name;
```

### Step 3. 장례식장 선택

| 항목 | 설명 |
|------|------|
| UI | 카드 리스트 (스크롤) |
| 카드 정보 | 장례식장명, 유형(병원/전문), 공설/사설, 빈소 수, 주차 대수, 주소 |
| 정렬 | 빈소 수 내림차순 (규모순) |
| 검색 | 장례식장명 검색 필터 |

**카드 UI 구성:**
```
┌────────────────────────────────────┐
│  고려대구로병원장례식장               │
│  병원 · 사설 · 직영                 │
│  빈소 11실 · 주차 945대             │
│  서울특별시 구로구 구로동로 148       │
│                         [선택하기]  │
└────────────────────────────────────┘
```

**API 쿼리:**
```sql
SELECT facility_cd, company_name, funeral_type,
       public_label, manage_class,
       mortuary_count, parking_count, full_address,
       facility_fees
FROM funeral_halls
WHERE gungu_cd = :selected_gungu
ORDER BY mortuary_count DESC;
```

### Step 4. 빈소 규모 선택 + 예상 조문객 수 입력 (3일장만)

> **무빈소 선택 시 이 Step은 건너뛰고 Step 5로 직행한다.**

장례식장의 `facility_fees` 데이터에서 빈소 정보를 추출하여 분기.

#### 4-A. 면적 데이터가 있는 경우 (평수_㎡ 존재)

면적 기반으로 규모 자동 분류 후 사용자에게 규모 선택 UI 제공.

**규모 분류 기준:**

| 규모 | 면적 (㎡) | 평형 | 동시 수용 | 3일 총 예상 조문객 | 추천 대상 |
|------|----------|------|---------|----------------|---------|
| 소형 | 60~100 | 20~35평 | 15~25명 | 50~100명 미만 | 친척 중심 소규모, 가족장 |
| 중형 | 100~198 | 40~60평 | 30~50명 | 100~150명 | 일반적인 3일장, 직장인/일반인 |
| 대형 | 200~300 | 70~90평 | 60~80명 | 150~200명 | 인맥이 넓거나 사회적 활동 활발 |
| 특실 | 300~390 | 100~130평 | 100명 이상 | 200~300명 | 기업장, 단체장, 문상객 집중 |
| VIP실 | 390 이상 | 130평 이상 | 100명 이상 | 300명 이상 | VIP, 정재계 인사, 대규모 기업장 |

**면적 → 규모 매핑 로직:**
```ts
function classifySize(sqm: number): string {
  if (sqm < 100) return 'small';
  if (sqm < 200) return 'medium';
  if (sqm < 300) return 'large';
  if (sqm < 390) return 'premium';
  return 'vip';
}
```

**UI: 규모 선택 카드**
```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│    소형       │ │    중형       │ │    대형       │ │    특실       │ │   VIP실      │
│  20~35평     │ │  40~60평     │ │  70~90평     │ │  100~130평   │ │  130평+      │
│  수용 15~25명 │ │  수용 30~50명 │ │  수용 60~80명 │ │  수용 100명+  │ │  수용 100명+  │
│  조문 ~100명  │ │  조문 ~150명  │ │  조문 ~200명  │ │  조문 ~300명  │ │  조문 300명+  │
└─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

규모 선택 시 → 해당 규모에 속하는 빈소의 `facility_fees` 항목을 필터링하여 시설 사용료 노출.

### Step 5. 예상 비용 결과 화면

모든 선택이 완료되면 항목별 예상 비용을 산출하여 결과 카드로 노출.

#### 5-1. 비용 항목 구성

**3일장 선택 시:**

| 구분 | 항목 | 금액 산출 방식 |
|------|------|-------------|
| 시설 사용료 | 빈소 사용료 | 선택한 장례식장의 실제 데이터 (3일분) |
| 시설 사용료 | 입관실 사용료 | 선택한 장례식장의 실제 데이터 |
| 부대비용 | 고인 이송비 | 수도권/비수도권 평균가 |
| 부대비용 | 안치실 안치료 | 수도권/비수도권 평균가 (x 3일) |
| 식대 | 음식비 | 수도권/비수도권 평균 단가 x **입력한 예상 조문객 수** |
| **합계** | | **전체 항목 합산** |

**무빈소 선택 시:**

| 구분 | 항목 | 금액 산출 방식 |
|------|------|-------------|
| ~~시설 사용료~~ | ~~빈소 사용료~~ | ~~제외~~ |
| 부대비용 | 고인 이송비 | 수도권/비수도권 평균가 |
| 부대비용 | 안치실 안치료 | 수도권/비수도권 평균가 |

## 4. 예담 상품 추천 로직

예상 총 비용 기준으로 적절한 예담 상품을 자동 추천.

| 예상 총 비용 | 추천 상품 | 추천 사유 |
|------------|---------|---------|
| ~300만원 미만 | 예담 1호 (무빈소) | 소규모/간소 장례에 적합 |
| 300~500만원 | 예담 2호 | 가장 일반적인 3일장 규모 |
| 500~700만원 | 예담 3호 | 중대형 규모 장례 |
| 700만원 이상 | 예담 4호 | 대규모 장례, 프리미엄 서비스 |

**절약 금액 계산:**
```ts
const saving = estimatedTotal - yedamProductPrice;
// 예: 9,212,000 - 2,300,000 = 6,912,000원 절약

       + 예상 조문객 수 입력                                │
       │                                                 │
       ├── Case A: 면적 있음 → 규모 카드 선택               │
       └── Case B: 면적 없음 → 리스트 직접 선택             │
                                                         │
       ├── 무빈소 (Step 4 스킵) ─────────────────────────┘
       │
       ▼
[Step 5] 결과 산출
       ├── 시설 사용료: facility_fees 실제 데이터 (3일장만)
       ├── 부대비용: 수도권/비수도권 평균가 상수
       ├── 식대: 평균 단가 x 입력한 조문객 수 (3일장만)
       └── 예담 상품 추천: 총 비용 구간별 매칭
```
