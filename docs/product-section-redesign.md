# 장례상품 안내 섹션 리디자인

## 현재 문제점

1. **테이블 기반 레이아웃** — 모바일에서 가로 스크롤 필수 (`min-w-[700px]`, `min-w-[500px]`)
2. **전체 비교표** — 4개 상품 컬럼이 한 줄에 나열되어 좁은 화면에서 읽기 어려움
3. **상품별 상세** — 구분/필수항목/내용 3컬럼 테이블이 모바일에서 비효율적
4. **이미지 활용 부족** — `PRODUCT_LABEL_IMAGES`에 18종 이미지가 있지만 테이블 셀 안에 작게 표시

---

## 개선 방향

> 참고: 고이(goi) 플랫폼의 카드/리스트형 UI
> 가격 노출 안 함

### 핵심 컨셉: Sticky 탭 + 이미지 카드 그리드 + 세로 스크롤

- 테이블, 아코디언 모두 사용하지 않음
- 하나의 통일된 이미지 카드 컴포넌트로 전체비교 / 상품별 상세 모두 처리
- 카테고리별로 세로 스크롤하며 모든 정보 확인 (클릭/펼치기 불필요)

---

### 공통: Sticky 탭 바

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [ 전체비교 ]  [ 1호 ]  [ 2호 ]  [ 3호 ]  [ 4호 ]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 스크롤해도 상단에 고정 (sticky)
- 탭 전환으로 즉시 비교 가능 — 별도 비교표 없이도 "2호 보고 → 3호 보고" 가 자연스러움

### 카드 컴포넌트 설계

## A/B 뷰 전환 (개발 중 비교용)

기존 테이블 UI를 제거하지 않고, 키보드 단축키로 두 버전을 전환할 수 있게 처리.

| 단축키 | 동작 |
|--------|------|
| `Cmd + A` | **카드 UI (신규)** 표시 — 기본값 |
| `Cmd + B` | **테이블 UI (기존)** 표시 |

### 구현 방식

```tsx
const [viewMode, setViewMode] = useState<'card' | 'table'>('card');

useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if (e.metaKey && e.key === 'a') { e.preventDefault(); setViewMode('card'); }
    if (e.metaKey && e.key === 'b') { e.preventDefault(); setViewMode('table'); }
  };
  window.addEventListener('keydown', handler);
  return () => window.removeEventListener('keydown', handler);
}, []);
```

- `viewMode === 'card'` → 새로운 이미지 카드 그리드 UI 렌더링
- `viewMode === 'table'` → 기존 테이블 UI 렌더링 (코드 그대로 유지)

## 구현 범위

### 변경 파일

| 파일 | 변경 내용 |
|------|----------|
| `general-funeral.tsx` | 섹션 11에 카드 UI 추가, 기존 테이블 UI 유지, `viewMode` state로 분기 |
| `constants.ts` | 데이터 구조 유지, `PRODUCT_LABEL_IMAGES` 키 1건 추가 (`유골함/횡대(매장시)`) |

### 기존 테이블 코드

- 삭제하지 않음
- `viewMode === 'table'` 조건으로 감싸서 `Cmd + B`로 전환 시 표시

### 신규 카드 UI 코드

- `viewMode === 'card'` 조건으로 감싸서 기본 표시
- `ProductItemCard` 컴포넌트 (`mode="compare" | "single"`)
- 카테고리별 이미지 카드 그리드

### 유지 대상

- 상품 탭 버튼 (전체비교 / 1호 / 2호 / 3호 / 4호) → sticky로 변경 (양쪽 모드 공통)
- 추천 대상 리스트 (상품별 탭)
- `PRODUCT_LABEL_IMAGES` 이미지 데이터
- `noticeItems` 안내사항 아코디언
- `inquirySectionRef` ref 연결
