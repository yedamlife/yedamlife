## 지역별 평균 비용 (20만원 미만 기준)

> 이상치(20만원 이상) 항목을 제외한 실질적 시장가 기준

| 구분 | 건수 | 장례식장 수 | 평균 | 중간값 | 최소 | 최대 |
|---|---:|---:|---:|---:|---:|---:|
| 수도권 (서울·인천·경기) | 36건 | 13곳 | **69,222원** | 60,000원 | 25,000원 | 150,000원 |
| 수도권 외 | 34건 | 25곳 | **59,441원** | 40,000원 | 15,000원 | 150,000원 |

### 수도권 조회 쿼리

```sql
WITH cleaning_fees AS (
  SELECT
    h.facility_cd,
    h.sido_cd,
    (fee->>'요금')::int AS amount
  FROM funeral_halls h
  CROSS JOIN LATERAL jsonb_array_elements(
    COALESCE(h.facility_fees, '[]'::jsonb) || COALESCE(h.service_items, '[]'::jsonb)
  ) AS fee
  WHERE fee->>'판매여부' = 'Y'
    AND fee::text LIKE '%청소%'
    AND (fee->>'요금')::int > 0
    AND (fee->>'요금')::int < 200000          -- 20만원 이상 제외
    AND h.sido_cd IN ('6110000', '6280000', '6410000')  -- 서울·인천·경기
)
SELECT
  COUNT(DISTINCT facility_cd)::int                         AS hall_count,
  ROUND(AVG(amount))::int                                  AS avg_amount,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY amount)::int AS median_amount,
  MIN(amount)::int                                         AS min_amount,
  MAX(amount)::int                                         AS max_amount,
  COUNT(*)::int                                            AS sample_count
FROM cleaning_fees;
-- 기대: hall_count=13, avg_amount=69222, median_amount=60000
```

## 특이사항

- 카테고리 분포: `기타` 55건 / `시설임대료` 18건 / `안치실이용료` 2건
- 삼성서울병원 등 일부 대형 장례식장은 빈소 면적(㎡)별로 청소비를 세분화해 공시
- 면목녹색병원(400,000원)·춘해병원·고성영락원(각 300,000원)은 이례적으로 높은 편
- 강원대병원(15,000원)이 최저, 대구 지역 병원 장례식장은 30,000원으로 일정한 수준
- 지역별로는 **부산(10곳)·인천경기(10곳)**가 가장 많이 공시

## 통계 집계 SQL (Postgres / Supabase)

`funeral_halls` 테이블의 `facility_fees` / `service_items` JSONB 배열을 unnest해서  
**판매여부=Y** 이고 **요금 > 0** 이며 **항목명에 '청소' 포함**인 항목만 집계.

### 통계 쿼리

```sql
WITH cleaning_fees AS (
  SELECT
    h.facility_cd,
    (fee->>'요금')::int AS amount
  FROM funeral_halls h
  CROSS JOIN LATERAL jsonb_array_elements(
    COALESCE(h.facility_fees, '[]'::jsonb) || COALESCE(h.service_items, '[]'::jsonb)
  ) AS fee
  WHERE fee->>'판매여부' = 'Y'
    AND fee::text LIKE '%청소%'
    AND (fee->>'요금')::int > 0
)
SELECT
  COUNT(DISTINCT facility_cd)::int                         AS hall_count,
  ROUND(AVG(amount))::int                                  AS avg_amount,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY amount)::int AS median_amount,
  MIN(amount)::int                                         AS min_amount,
  MAX(amount)::int                                         AS max_amount,
  COUNT(*)::int                                            AS sample_count
FROM cleaning_fees;
```

### Supabase RPC 함수

```sql
CREATE OR REPLACE FUNCTION get_cleaning_stats()
RETURNS TABLE(
  hall_count int,
  avg_amount int,
  median_amount int,
  min_amount int,
  max_amount int,
  sample_count int
)
LANGUAGE sql STABLE AS $$
  WITH cleaning_fees AS (
    SELECT
      h.facility_cd,
      (fee->>'요금')::int AS amount
    FROM funeral_halls h
    CROSS JOIN LATERAL jsonb_array_elements(
      COALESCE(h.facility_fees, '[]'::jsonb) || COALESCE(h.service_items, '[]'::jsonb)
    ) AS fee
    WHERE fee->>'판매여부' = 'Y'
      AND fee::text LIKE '%청소%'
      AND (fee->>'요금')::int > 0
  )
  SELECT
    COUNT(DISTINCT facility_cd)::int,
    ROUND(AVG(amount))::int,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY amount)::int,
    MIN(amount)::int,
    MAX(amount)::int,
    COUNT(*)::int
  FROM cleaning_fees;
$$;
```
